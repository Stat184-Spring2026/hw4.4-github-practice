# Plan for HW #4.4

## Goal

Create a public GitHub repository that showcases my HW #4.3 reproducible analysis project while demonstrating version control skills such as branching, issues, commits, and pull requests.

## Needs

To complete this project, I will need:

* My completed HW #4.3 Quarto file
* My submitted HW #4.3 PDF
* GitHub account
* Git or GitHub web interface
* README file
* Project images/plots
* Time to organize and document the repository

## Steps

1. Gather all files from HW #4.3.
2. Create a clean project folder structure.
3. Create a public GitHub repository.
4. Upload the QMD, PDF, README, and images.
5. Create at least one additional branch besides main.
6. Make improvements on the branch.
7. Open and merge a pull request.
8. Create at least two GitHub Issues.
9. Use clear commit messages throughout the project.
10. Review the repository for organization and completeness.
11. Submit the GitHub repository URL.

## HW #4.3 Maintenance Plan

1. Keep source files organized and clearly named.
2. Preserve reproducibility by storing the original QMD file.
3. Keep final outputs (PDF and figures) available.
4. Update README if project files change.
5. Continue improving GitHub and documentation skills.

## Success Criteria

The repository will be complete when it contains all required files and demonstrates proper GitHub workflow features.
